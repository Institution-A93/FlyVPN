# ADR-0017: DNS на egress — резолвер sing-box, без unbound

- Статус: accepted
- Дата: 2026-06-12

## Контекст
ADR-0011 предполагал на egress отдельный unbound (рекурсивный резолвер на localhost),
куда sing-box направляет DNS клиентов. При боевом деплое unbound на узле стабильно не
запускался (служба падала; точная причина — порт/anchor/systemd-resolved — без диагностики
на узле). unbound на egress оказался необязательным компонентом.

## Решение
Убираем unbound с egress. DNS клиентов резолвит **сам sing-box** через публичный резолвер
(`egress_dns_upstream`, по умолчанию `1.1.1.1`) outbound'ом `direct`. Запрос уходит из сети
egress → резолв происходит на egress, не утекает из РФ. Цель ADR-0011 (резолв на egress)
сохранена; меняется лишь механизм (sing-box dns вместо unbound).

Это уточняет ADR-0011 (пункт про unbound) и упрощает egress-роль (минус компонент и целый
класс падений при старте).

## Следствия
- egress-роль: задачи unbound удалены; залипший unit нейтрализуется (stop/disable/mask).
- sing-box `dns.servers` → публичный резолвер через `direct`.
- README §2.3 и egress-доки приведены в соответствие.
