# ADR-0009: Только OSS-компоненты; IaC-тул — OpenTofu

- Статус: accepted
- Дата: 2026-06-09

## Контекст
Принцип проекта: в стеке используем только OSS-компоненты. Это затрагивает уже
зафиксированные решения, где инструмент назван «Terraform»: с версии 1.6 HashiCorp
перевела Terraform на BUSL — это не OSS-лицензия.

## Решение
1. В стек входят только OSS-компоненты.
2. IaC-инструмент — **OpenTofu** (MPL-2.0, Linux Foundation), drop-in замена
   Terraform: тот же HCL/`.tf`, те же провайдеры (`hcloud`, `selectel`). CLI — `tofu`.
   Ранее зафиксированные «Terraform»-упоминания (ADR-0002/0003/0004) читать как OpenTofu.
3. Каталог `infra/terraform` сохраняет имя (де-факто термин для HCL); меняется только
   инструмент, не формат файлов.

## Известные конфликты к разрешению
- **GeoDNS** (README §2.6/§3): Cloudflare Load Balancing / NS1 — проприетарные SaaS.
  На phase 2 заменить на OSS-вариант (напр. PowerDNS + собственные health-checks
  оркестратора). На MMVP не блокирует (один узел, без GeoDNS-failover).

## Последствия
Совместимость провайдеров и модулей сохраняется. Брендовая привязка к проприетарному
Terraform убрана. Любой новый компонент стека проверяется на OSS-лицензию до внесения.
