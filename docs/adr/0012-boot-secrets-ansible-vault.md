# ADR-0012: Boot-секреты узлов — ansible-vault на MMVP, оркестратор на phase 2

- Статус: accepted
- Дата: 2026-06-09

## Контекст
Роль egress требует секреты (Reality private key, клиентские UUID), которых нет в
репозитории. Нужен способ подать их при прогоне Ansible. Целевая модель (README §2.6,
ADR-0002): секреты генерирует и выдаёт оркестратор при старте узла, на узле они не
персистятся. Но оркестратор ещё не реализован.

## Решение
- **На MMVP**: boot-секреты лежат в `group_vars/<group>/vault.yml`, зашифрованном
  `ansible-vault`. Файл в `.gitignore` (паттерн `group_vars/*/vault.yml`), в репозитории
  только `*.example`. Пароль vault — вне репозитория.
- **На phase 2**: значения генерирует/ротирует оркестратор и подаёт узлу в рантайме;
  vault-файлы выводятся из обихода. Контракт переменных роли (`egress_reality_*`,
  `egress_clients`) не меняется — меняется только источник.

## Последствия
Старт без оркестратора возможен (ручной vault), при этом инвариант «секретов в репо нет»
соблюдён. Переход на оркестратор не затрагивает роли — только механизм доставки.
