# ADR-0013: Язык backend-сервисов — открытое решение

- Статус: proposed
- Дата: 2026-06-09

## Контекст
В исходной спеке (README §7) и в инвариантах CLAUDE.md язык backend-сервисов
(`config-api`, `orchestrator`) был указан как «Go». При ревью выяснилось: это была
гипотеза из спецификации, а не зафиксированное требование. Преждевременно вшивать её в
инварианты — ошибка (требование-галлюцинация).

## Решение
Язык backend-сервисов **не фиксируется** на данном этапе. «Go» убран из инвариантов
и описаний; статус решения — `proposed`, выбор отложен.

Критерии для будущего выбора (когда дойдём до сервисов):
- простой деплой на cattle-узлы (предпочтительно статический бинарь);
- зрелые библиотеки для HTTP, PostgreSQL, bcrypt, генерации plist/XML;
- скорость разработки MMVP vs строгость/производительность;
- только OSS-тулчейн и библиотеки (ADR-0009).

Кандидаты к рассмотрению: Go, Rust, Python (не исчерпывающе).

## Следствия
- Работа продолжается с языко-независимых частей: схема PostgreSQL (plain SQL),
  контракты API, шаблон .mobileconfig.
- Раннер миграций тоже не привязан к языку: миграции — обычные `.sql` (up/down),
  совместимые с разными тулами (golang-migrate, dbmate, Flyway, psql).
- Решение будет переведено в `accepted` отдельным ADR при фактическом выборе.
