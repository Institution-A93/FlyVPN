# ADR-0001: Граница MMVP

- Статус: accepted
- Дата: 2026-06-08

## Контекст
MVP из README сам по себе уже урезан. Нужен ещё более тонкий первый срез (MMVP),
доказывающий, что сквозной путь пакета работает и продукт можно отдать живому
юзеру. После MMVP ожидаем, что реальный опыт скорректирует требования.

## Решение
В MMVP входит минимальный сквозной путь:
- по одному узлу каждой роли: 1 RU ingress, 1 foreign egress, 1 control plane;
- умный ASN-split (см. ADR-0006);
- auth через RADIUS/EAP-MSCHAPv2 (см. ADR-0005);
- Plati-интеграция (см. ADR-0007);
- IaC для обеих ролей узлов с самого начала (см. ADR-0002).

Вне MMVP: multi-node GeoDNS-failover, per-user выбор локации, FDE remote unlock,
авто-ротация секретов, Prometheus/Grafana — как в §6 README.

## Последствия
Быстрее до «первая транзакция → рабочий VPN». Failover на старте — ручной.
