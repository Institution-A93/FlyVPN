# ADR-0005: Auth через RADIUS / EAP-MSCHAPv2

- Статус: accepted
- Дата: 2026-06-08

## Контекст
Ключевое требование — на RU-узле нет юзерских данных (узел могут изъять). Сам RADIUS
не самоцель. Рассматривали альтернативу EAP-TLS + CA без RADIUS: узел держит только
CA, но отзыв (CRL/OCSP) и аккаунтинг пришлось бы строить отдельно.

## Решение
Берём RADIUS/EAP-MSCHAPv2. strongSwan на ingress — RADIUS-клиент, FreeRADIUS +
PostgreSQL на control plane за границей держат юзеров. Это выносит auth И accounting
за границу и даёт sticky-IP (`Framed-IP-Address`) из коробки. Жизненный цикл юзера
(Plati создаёт/продлевает/отзывает) ложится 1:1 на FreeRADIUS+Postgres.

## Последствия
На узле запускается RADIUS-клиент/прокси к control plane через туннель. Решение
обратимо: стык auth держим так, чтобы EAP-TLS можно было ввести позже без переделки
узла целиком.
