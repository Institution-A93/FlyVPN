# ADR-0002: Реальное облако с самого старта

- Статус: accepted
- Дата: 2026-06-08

## Контекст
Рассматривали local-first (Docker Compose) против реального облака. Ротация узлов —
ядро продукта (RU ingress ловит targeted-блок ТСПУ), значит IaC должен бить в живые
API провайдеров, а не в моки.

## Решение
Разворачиваем MMVP сразу в реальном облаке. IaC (OpenTofu) — first-class для обеих
ролей узлов, не только egress. Узлы — cattle, не pets: ротация = `tofu apply`
нового узла + регистрация в оркестраторе + дренаж старого.

## Последствия
Дороже и медленнее в отладке, чем локально. Зато ротация и провижн отлажены на
реальном пути с первого дня. Сейчас в наличии только аккаунт Hetzner; RU-аккаунт
(Selectel) и домен добываются по ходу — до их появления в реальном облаке живут
egress + control plane, ingress подключается следом.
