# ADR-0010: Лицензия проекта — AGPL-3.0

- Статус: accepted
- Дата: 2026-06-09

## Контекст
Проект коммерческий и публичный (open source). Цель владельца: пусть появляется
больше таких сервисов (распространение по миссии анти-цензуры не ограничиваем), но
улучшения кода не должны закрываться — в том числе когда форк крутят как сетевой
сервис. Moat бизнеса — операционка (живые узлы, ротация, оплата, поддержка), а не
секретность кода.

Лицензии стороннего стека совместимы: наш код (config-api, orchestrator, HCL,
Ansible) — отдельные процессы, оркеструющие сторонние демоны (strongSwan/FreeRADIUS
GPLv2, Xray/OpenTofu MPL-2.0, unbound/nginx/PostgreSQL BSD-like). Это агрегация, не
линковка — конфликта с выбором AGPL для нашего кода нет.

## Решение
Лицензия проекта — **AGPL-3.0** (GNU Affero General Public License v3.0).
- Запуск/развёртывание/форк свободны → «больше таких сервисов».
- §13 закрывает SaaS-лазейку: кто модифицирует код и отдаёт его по сети — обязан
  опубликовать изменения → «улучшения не закрывают».

Рассмотренные альтернативы:
- Apache-2.0 / MIT — позволяют закрыть улучшения (не подходит).
- GPL-3.0 — SaaS-лазейка: модификации в сервисе можно не открывать.
- MPL-2.0 — копилефт по файлам без сетевой клаузы (улучшения в сервисе можно не отдавать).

## Граница (важно)
AGPL открывает КОД. Боевые операционные значения, полезные цензору, остаются
приватными (вне репозитория): IP узлов, актуальные SNI-cover домены Reality,
конкретные анти-DPI параметры, любые секреты. Открыта логика — не «карта для ТСПУ».

## Последствия
- В корне — файл `LICENSE` с полным текстом AGPL-3.0.
- Новые файлы исходников при желании снабжаются SPDX-заголовком
  `// SPDX-License-Identifier: AGPL-3.0-only`.
- При приёме контрибуций учитываем требования AGPL (возможен CLA/DCO — решить позже).
